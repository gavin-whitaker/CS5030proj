KEY FINDINGS
------------
1. OpenMP delivers modest speedup (1.4×) limited by:
   - Per-iteration synchronization barriers
   - Serial centroid merge phase
   - Cache contention at high thread counts

2. CUDA is comparable to best OpenMP (1.4–1.5×) for this dataset:
   - PCIe centroid transfer overhead dominates improvement
   - Block size has negligible impact (compute not memory-bound)
   - Much simpler to tune than OpenMP for this workload

3. MPI outperforms OpenMP significantly (2.33× vs 1.42× at 4P vs 16T):
   - Distributed memory avoids cache contention
   - Centroid communication (480 bytes/iter) is negligible
   - Scales better due to Amdahl's Law: larger serial fraction in OpenMP

4. MPI+CUDA is the overall best (2.73× speedup):
   - GPU handles assignment; MPI handles centroid sync
   - Diminishing returns beyond 2 ranks for K=10 (latency-dominated)
   - Larger K (e.g., K=50) would show better GPU/MPI scaling

5. All implementations validated against serial (0 mismatches):
   - OpenMP: PASS (direct + remapped comparison)
   - CUDA:   PASS (direct + remapped comparison)
   - MPI:    PASS (remapped comparison — label order may differ)
   - MPI+CUDA: PASS (remapped comparison)

RECOMMENDATION FOR PRODUCTION
------------------------------
  - Single node, no GPU: OpenMP 8–16T (2.86–2.59 s)
  - Single node, with GPU: CUDA bs=256 (2.65 s) or MPI+CUDA 1P (1.61 s)
  - Multi-node, CPU only: MPI 4P (1.58 s)
  - Multi-node, with GPUs: MPI+CUDA 4P (1.35 s) — BEST OVERALL
  - For K >> 10: MPI+CUDA scaling advantage increases substantially
