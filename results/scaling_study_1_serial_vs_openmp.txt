ANALYSIS
--------
Speedup is modest (1.3–1.4× on 16 cores) due to:
  - Fine-grained synchronization: implicit barrier after each parallel region
    means all threads must wait at the boundary of every iteration (50 barriers
    for 50 iterations)
  - Centroid reduction overhead: thread-private accumulators are merged serially
    after each assignment phase; this serial merge does not scale with threads
  - Memory contention: all threads read the shared centroids array each
    iteration; at 16 threads, cache-line contention for the centroid reads
    limits further improvement
  - Small per-thread workload: at 16 threads each thread processes only ~75K
    points; thread creation/scheduling overhead is non-trivial relative to work

Efficiency drops sharply with thread count — from 98% (1T) to 9% (16T) —
consistent with Amdahl's Law applied to the serial merge phase.

The 1-thread OpenMP time (3.76 s) closely matches serial (3.68 s), confirming
OpenMP overhead is minimal at single-threaded execution.

CONCLUSION: OpenMP provides 1.4× speedup at 16 threads. For single-node
workloads, CUDA (2.5 s) is substantially more efficient than OpenMP parallelism.
